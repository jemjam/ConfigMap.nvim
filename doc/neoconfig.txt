*neoconfig*

neoconfig.nvim	Quick loader for keymaps, commands, functions, and autocmds

NAME
	neoconfig - lightweight declarative loader for Neovim config items

SYNOPSIS
	Use `:lua require('neoconfig').setup(opts)` or call from your plugin manager's `config`.

DESCRIPTION
	`neoconfig` accepts a single table (or a function that returns a table) and
	creates keymaps, user commands, autocmds, and validates simple named Lua
	functions. It's intentionally minimal and has no UI or external dependencies.

TOP-LEVEL FIELDS
	- `keymaps`  : list of keymap entries (see examples)
	- `commands` : list of user command entries
	- `funcs`    : list of named Lua functions (validated at setup)
	- `autocmds` : list of autocmd definitions
	- `defaults` : per-kind defaults merged into each item (optional)

EXAMPLES

Lazy.nvim (recommended pattern)
	local spec = {
	  'jemjam/neoconfig.nvim',
	  opts = {
	    defaults = { keymaps = { silent = true, noremap = true } },
	    keymaps = {
	      { lhs = '<leader>ff', rhs = ':Telescope find_files<CR>', mode = 'n', desc = 'Find files' },
	    },
	    commands = {
	      { name = 'SayHello', handler = function() print('hello') end, opts = { desc = 'Say hello' } },
	    },
	  },
	  config = function(_, opts) require('neoconfig').setup(opts) end,
	}

Direct require usage
	require('neoconfig').setup({
	  keymaps = {
	    { lhs = '<leader>p', rhs = ':Telescope live_grep<CR>', mode = {'n','v'}, desc = 'Live grep' },
	  },
	  autocmds = {
	    { events = 'BufWritePre', pattern = '*', callback = function() vim.lsp.buf.format() end, opts = { once = false } },
	  },
	})

NOTES
	- `setup` accepts either a table or a function returning the table.
	- Duplicate keymaps (same mode + lhs) or duplicate command/func names will raise an error.
	- This module does not expose functions globally â€” it only validates and registers
	  the provided definitions with Neovim.
